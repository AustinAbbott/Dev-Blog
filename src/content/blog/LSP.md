Somehow, I missed the automated install suggestion for LSP in Claude Code, so I'm writing this post for anyone else that missed the boat that [launched in December](https://github.com/anthropics/claude-code/blob/main/CHANGELOG.md#2074).

For the uninitiated, enabling LSP (Language Server Protocol) provides the same code intelligence tools bundled with VS Code (e.g., `goToDefinition`, `findReferences`) in the terminal, which, in my experience, makes Claude Code absolutely fly. It also uses way fewer tokens since Claude doesn't have to grep as much and lowers the bar for which tasks cheaper models can handle. As an example, I was able to have Haiku accurately detect and remove all unused code within a legacy directory in one shotâ€”something Opus couldn't do without LSP.

Weirdly, the [Claude Code docs on this](https://code.claude.com/docs/en/discover-plugins#code-intelligence) aren't super helpful, so here's a quick setup guide for TypeScript LSP:

1. Install the TypeScript language server -> `npm install -g typescript-language-server typescript`
2. Enable LSP in your terminal config file (using nano + .zschrc)
   - `nano ~/.zshrc`
   - Add `export ENABLE_LSP_TOOLS=1`, save, exit
   - Run `source ~/.zshrc` to refresh your terminal(s) (or close and reopen)
3. Enable LSP in Claude Code
   - Run `claude`
   - Using the `/plugins` command, install "typescript-lsp"  (claude-plugins-official)
   - Restart Claude
